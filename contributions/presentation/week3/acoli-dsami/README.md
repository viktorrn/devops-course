## Description

This presentation will present the fundamental differences between continuous deployment and continuous delivery, two practices often confused but with very different implications for software teams. It will also include a simple pipeline that decides whether a PR is suitable for auto-deploy or needs manual approval based on some specific metric, such as how core the change is to the project. 

**Relevance**

Many organizations don't differentiate between this two practises. By clarifying the distinction, teams can make more informed decisions about their release process, whether to prioritize control or velocity.
