## Description

We want to present about GitOps.
This presentation would give a brief overview of CD and challenge, and touch upon how and why the concept of GitOps was created. Overview how GitOps would be implemented from a process perspective and the benefits of using it. Will then compare the technical components of traditional CD with GitOps. Then we will give a code example how to change from a conventional CD to GitOps.

The layout would be something like this,
- Introduce how code goes from a dev to deployment
- Walk through how CD improves this process
- Discuss potential issues with CD
- Introduce GitOps and why it is highly suitable for a working CD
- Walk through how GitOps solves some potential issues with GitOps
- We would then show some code comparing traditional CD tool with a more suitable code configuration for GitOps
- Make some sort of a conclusion about the adoption of GitOps

**Relevance**

CD is a crucial aspect of GitOps and vice-versa as GitOps solves/manages one of the primary issues with CD which is configuration version control and failure management.
